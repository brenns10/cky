Adjust the FSM according to its modifier, if any.

When a character, character class, or parenthesized regex is read in, it could be followed by the modifiers `*`, `+`, or `?`. This function adjusts the FSM for those modifiers, and adjusts the location pointer if one was present.

fsm* regex_parse_recursive ( const wchar_t *  regex,
const wchar_t **  final 
)

The workhorse recursive helper to regex_parse().

Crawls through a regex, holding a FSM of the expression so far. Single characters are concatenated as simple 0 -> 1 machines. Parenthesis initiate a recursive call. Pipes union the current regex with everything after.

Needs to be called with a non-NULL value for both, which is why there is a top level function.

Parameters:
regexLocal pointer to the location in the regular expression.
finalPointer to the parent's location.
Returns:
An FSM for the regular expression.
