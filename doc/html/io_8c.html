Print an FSM to graphviz dot format.

Graphviz is a neat CLI tool to create diagrams of graphs. The dot format is an input language for graphs. This function takes a FSM and prints it to a file as a dot file, so that it can then be converted to a image by graphviz.

fsm* fsm_read ( const wchar_t *  source)

Read a FSM from a string! The format is as follows: 

   START LINE (start: n)
   ACCEPTING LINES ... (>=1) (accept: n)
   TRANSITION LINES (X-Y:[+|-]A-B[[A-B]...])
See also:
fsm_read_get_int() for info on START LINE and ACCEPTING LINES
fsm_read_trans() for info on the TRANSITION LINES

Note that this function produces heavy output on stdout if compiled with SMD_ENABLE_DIAGNOSTIC_PRINTING.

Parameters:
sourceThe string source
Returns:
A FSM from the string if possible, NULL on error.

Get an integer value from a key-value line.

The line is in the form "key: value ", where whitespace is not allowed in the key section, but is allowed for the value section. The value must be a positive decimal integer. On error, returns -1.

Parameters:

fsm_trans* fsm_read_trans ( const wchar_t **  source,
int *  start 
)

Reads a transition from a line.

This function reads a transition from a line. The transition can have an arbitrary number of ranges, each separated by a space. The line should be of the form: `X-Y:[+|-]A-B[[A-B]...]`, where `X` and `Y` are state numbers, `A` and `B` are characters forming a range, and the `+` or `-` determine whether the transition is positive (only those in the range) or negative (everything not in that range). Note that there is only one `+` or `-`; it applies for all ranges in the transition.

See also:
SFIRSTSTATE etc. for more info on the meaning of the states.
Parameters:
sourceThe source pointer. It is advanced to the beginning of the next line.
[out]startWhere to put the start state of the transition.
Returns:
The transition.
